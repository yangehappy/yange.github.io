史上最简单的hexo+github搭建个人网站教程

好吧，我承认我标题党了。自从去年夏天搭好了这个网站，给不少人安利过hexo,并成功的把他们带进了坑了。最近越发觉得有必要写个简单教程，大神请自动略过，本教程主要针对小白。
就像造房子需要先准备钢筋水泥等原材料一样，使用hexo搭建个人站点需要的原材料是nodejs+git+github。因为hexo是基于nodejs的，所以nodejs是必须要安装的。git是个版本管理软件，我们可以用它来管理我们的hexo文件，平时的维护和博客文章的上传都需要用到它。即使你对他们都不了解，没关系，安装好nodejs和git后只要按照以下步骤就可以搭建好一个个人站点了。这里就不介绍nodejs和git的安装了，不知道怎么下载安装的同学请自行谷歌。

本地操作

假设现在你的电脑上装好了nodejs和git，在电脑上任意磁盘新建一个文件夹(名字可以按自己喜好来命名，假设这里我们命名为myblog)，这个文件夹用来存放我们搭建的hexo博客文件。双击打开myblog文件夹，进入后右键单击选择git bash,打开后是下面这样子的，它是git的命令行操作界面。

接着在git中输入npm install -g hexo安装hexo，等待一段时间后出现$和跳动的光标代表安装完成。输入hexo -v查看安装的hexo版本

ok，下面接着键入hexo init来初始化myblog文件夹，当出现下图所示界面代表hexo博客在本地安装完成

键入hexo s，在本地生成页面，提示在浏览器输入http://localhost:4000/查看博客页面，注意如果你电脑中的其他的程序占用了你的4000端口，你就无法打开网页，需要手动修改端口。hexo默认安装的是landscape主题，所以在浏览器中打开是这样的，当你看到这个页面，恭喜你，你已经成功的在本地搭建起了自己的第一个博客。

github远程操作

下面开始github上的操作，首先新建一个Repository，并命名为username.github.io，因为github默认一个账号只能绑定一个GitHub Pages，所以仓库的命名只能按github规定的格式来写，即你的github用户名.github.io。进入该仓库，找到Settings选项，点进去找到GitHub Pages，会看到 Your site is published at https://yangcong1.github.io/，意思是你的github pages启用了，并且部署在上面的网站。

返回本地操作

找到myblog目录下的_config.yml文件，并修改最后面的deploy，将其修改为

deploy:
type: git
repository: https://github.com/username/username.github.io
branch: master

修改完后执行命令:

npm install hexo-deployer-git - -save

接着使用hexo d命令将本地文件部署到github仓库。

完成

现在在浏览器中输入https://yourname.github.io/就可以访问你的博客了。
